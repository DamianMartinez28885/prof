Seccion de preguntas sobre HTTP

  1. ¿Sobre que protocolo binario está montado el protocolo HTTP?

    2. HTTP está montado sobre TCP que es un protocolo que se encarga de controlar y asegurar que todo el envío de datos llegue a su destino. Para establecer una conexión, el protocolo TCP genera un saludo de tres vías entre el cliente y el servidor. A su vez, TCP está montado sobre el protocolo IP.

  2. Cuáles son los clientes http y los servidores http mas utilizados

    3. Los principales clientes son Google, Mozilla y Microsoft.

    Los servidores: apache, Nginx, litespeed web server.

  3. ¿Qué verbos admite un comando en el requerimiento HTTP?

    4. Algunos de los métodos:
    CONNECT: Lo usan los proxys
    GET: Utilizado para la lectura (los valores viajan en la URL)
    PUT: Utilizado para crear recursos de un servidor
    DELETE: Utilizado para borrar recursos en un servidor
    POST: Utilizado mayormente para actualizar recursos en un servidor (los valores viajan en el body)

  4. ¿Qué contenido lleva el body de un requerimiento HTTP?

    5. El body de un requerimiento puede contener datos en cualquier tipo de formato, audio, video, html, datos ingresados a través de un formulario, archivos enviados en un proceso de upload, entre otros.

  5. ¿Qué diferencia existe entre un URL y un URI?

    6. La url es el dominio al cual quiero acceder, es la direccion del servidor al cual quiero conectarme y la uri es la direccion de los recursos a los cuales quiero acceder dentro del servidor.

  6. ¿Cómo se almacena información relacionado con las respuestas HTTP?

    7. Existen tres maneras en las que se puede manejar la informacion con las respuestas http:
    Sin cache: el navegador no almacena las respuestas, si no que estas se cargan totalmente en cada requerimiento.
    Cache con fechas: el navegador hace una peticion con header de if-Modified-Since y la fecha que tiene cargada, si el servidor no tiene una modificacion desde esa fecha entonces el navegador carga desde el cache el contenido.
    Igual que el caso anterior pero utilizando un identificativo (Etag) asociado a cada file en el servidor. Este es una etiqueta que se modifica en el servidor cada vez que se modifica el recurso. Si el cliente tiene almacenado el recurso en cache envía como header de req. el etag asosciado. Lo que se compara entonces en el server es este header con el etag del recurso. Si no fue modificado entonces solo devuelve una respuesta con un header «not Modified»

  7. ¿Qué significa Virtual Hosting?

    8. El virtual hosting se refiere a la práctica de hospedar múltiples dominios (sitios web) en un único servidor. Un solo servidor físico aloja los sitios web de varios clientes, cada uno con su propio dominio, pero compartiendo los recursos del servidor como la CPU, la memoria RAM y el espacio en disco.

  8. ¿ Porque el HTTP se considera STATE LESS?

    9. HTTP se considera "stateless" (sin estado) porque cada solicitud HTTP que realiza un cliente al servidor se procesa de forma independiente, sin tener en cuenta solicitudes anteriores

  9. ¿Cómo va a mejorar la velocidad de la WEB en poco tiempo?

    10. HTTP/2 es un protocolo binario que conserva la misma semántica que el protocolo HTTP1.1 compactando cabeceras, priorizando elementos a transferir y multiplexando sin esperar respuestas. Esto que significa que todos los verbos, cabeceras, etc. siguen funcionando sin cambios. De hecho, HTTP/2 busca resolvery compensar los problemas de velocidad quetiene la comunicaciónatravésTCP (lacapa de transporte dentro del protocolo HTTP).

  10. ¿Como va a mejorar el comportamiento de una aplicación WEB en ambientes ruidosos o de débil conexión?

    11. HTTP/3 mejora al http/2 en velocidad de establecimiento de conexión y en mayor capacidad de recuperación y velocidad en casos de estar en zonas con mucho ruido y en consecuencia de poca confiabilidad en los medios
